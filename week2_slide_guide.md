## หมวดที่ 1: ปูพื้นฐานความเข้าใจเรื่องพิกเซลและโครงสร้างภาพดิจิทัล

### สไลด์ที่ 1: ภาพดิจิทัลในสายตาคอมพิวเตอร์คืออะไร?
*   **โครงสร้างการจัดวางหน้าสไลด์ (Visual Layout):**
    *   ฝั่งซ้าย: รูปภาพผลไม้สีสดใสงดงาม 1 รูป
    *   ฝั่งขวา: การซูมขยายภาพผลไม้นั้นจนเห็นเป็นกล่องสี่เหลี่ยมทีละช่องชัดเจน (Pixels) และแสดงแถบตัวเลข (0-255) กำกับอยู่ในกล่องเหล่านั้น
*   **เนื้อหาบนสไลด์ (Slide Content):**
    *   **พิกเซล (Pixel - Picture Element):** หน่วยย่อยที่เล็กที่สุดของรูปภาพดิจิทัล
    *   คอมพิวเตอร์มองไม่เห็นภาพเหมือนมนุษย์ แต่มองเห็นเป็น **"ตารางตัวเลข 2 มิติ หรือ 3 มิติ" (Matrix / Array)**
    *   ค่าในแต่ละกล่องสี่เหลี่ยมคือค่าความสว่างของแสง ยิ่งตัวเลขมาก ยิ่งสว่างมาก
*   **บทบรรยายประกอบการสอนอย่างละเอียด (Teacher's Script & Review Note):**
    *   *"นักศึกษาครับ ลองจินตนาการถึงภาพปักครอสติช (Cross-stitch) หรือภาพที่วาดบนกระดาษกราฟ ทุกช่องสี่เหลี่ยมเล็ก ๆ ที่เราเล็งระบายสีลงไปนั่นแหละครับที่เรียกว่า พิกเซล (Pixel) เมื่อมองระยะไกล สายตาเราจะเบลอขอบตารางเหล่านั้นจนประกอบกันกลายเป็นภาพสวยงาม แต่สำหรับคอมพิวเตอร์ มันไม่มีตาเนื้อ มันมองเห็นเพียงกล่องสี่เหลี่ยมเหล่านี้ที่เรียงกันเป็นแถวและคอลัมน์ และในแต่ละกล่องจะมีตัวเลขระบุความสว่างของสีนั้น ๆ ไว้"*

### สไลด์ที่ 2: เจาะลึกแชนเนลสี BGR (ทำไม OpenCV ไม่ใช้ RGB?)
*   **โครงสร้างการจัดวางหน้าสไลด์ (Visual Layout):**
    *   ภาพแผ่นกระจกสี 3 แผ่นประกบกันซ้อนกัน (สีน้ำเงิน B, สีเขียว G, สีแดง R)
    *   แผนภูมิแสดงการประกบกระจก 3 ชั้น เพื่อฉายไฟออกมาเป็นสีสันต่าง ๆ
*   **เนื้อหาบนสไลด์ (Slide Content):**
    *   ภาพสีที่เราเห็นเกิดจากแม่สีแสง 3 สีผสมกัน: **B**lue (น้ำเงิน), **G**reen (เขียว), **R**ed (แดง)
    *   **แชนเนล (Channels):** ความลึกมิติที่ 3 ของอาเรย์ภาพภาพหนึ่ง มีขนาดเท่ากับ 3
    *   **กฎข้อสำคัญ:** OpenCV เรียงลำดับแชนเนลสีเป็น **BGR** (น้ำเงินก่อน แดงหลัง) ในขณะที่ซอฟต์แวร์อื่น เช่น PIL หรือ Matplotlib นิยมใช้ระบบ **RGB**
*   **บทบรรยายประกอบการสอนอย่างละเอียด (Teacher's Script & Review Note):**
    *   *"ทำไมต้อง BGR? เรื่องนี้เป็นเรื่องประวัติศาสตร์ครับ ในอดีตเมื่อหลายสิบปีทีก่อน ตอนที่ OpenCV ถูกคิดค้นขึ้นครั้งแรกโดย Intel รูปแบบสัญญาณโทรทัศน์และระบบกล้องส่วนใหญ่นิยมจัดเก็บในลำดับแชนเนล BGR ทางผู้พัฒนาก็จึงใช้รูปแบบนั้นมาจนถึงปัจจุบัน ดังนั้นเวลาเขียนโปรแกรมด้วย OpenCV ขอให้จำไว้เสมอว่าดัชนีช่องสีช่องแรก (ดัชนี 0) คือสีน้ำเงิน ดัชนีที่ 1 คือสีเขียว และดัชนีที่ 2 คือสีแดงครับ หากเรานำภาพที่อ่านจาก OpenCV ไปพล็อตกราฟตรง ๆ ด้วย Matplotlib โดยไม่แปลงสลับช่องสีก่อน สีแดงจะกลายเป็นสีน้ำเงิน และสีน้ำเงินจะกลายเป็นสีแดง ทำให้หน้าคนกลายเป็นคนตัวสีฟ้าเอเลี่ยนทันที"*

### สไลด์ที่ 3: ระบบพิกัดภาพ (Coordinate System) และจุดหลงลืมยอดนิยม (x, y) vs [y, x]
*   **โครงสร้างการจัดวางหน้าสไลด์ (Visual Layout):**
    *   แผนภูมิพิกัดเปรียบเทียบ 2 แบบขนานกัน:
        *   ด้านซ้าย: พิกัดเรขาคณิตคาร์ทีเซียนของกล้อง $(x, y)$ เริ่มต้นมุมซ้ายบน
        *   ด้านขวา: ดัชนีแถวและคอลัมน์ในตาราง NumPy `[row, col]` ซึ่งเท่ากับ `[y, x]`
    *   มีตารางตัวอย่างขนาด 3 แถว 4 คอลัมน์ และมีลูกศรชี้พิกัดแสดงให้เห็นความสับสนของจุด $(x=3, y=1)$ ซึ่งในการเข้าถึงอาเรย์จะเขียนว่า `img[1, 3]`
*   **เนื้อหาบนสไลด์ (Slide Content):**
    *   จุดกำเนิดพิกัด $(0, 0)$ อยู่ที่ **"มุมซ้ายบนสุดของภาพ"**เสมอ
    *   การอ้างอิงของไลบรารี OpenCV: ใช้รูปแบบเรขาคณิต `(กว้าง x, สูง y)` เช่นในฟังก์ชัน `cv2.resize` หรือคำสั่งวาดรูปทรง
    *   การอ้างอิงดัชนีผ่าน NumPy Array: ต้องอ้างอิงเป็นดัชนี `[แถว, คอลัมน์]` หรือ **`[y, x]`**
*   **บทบรรยายประกอบการสอนอย่างละเอียด (Teacher's Script & Review Note):**
    *   *"นี่คือจุดที่นักศึกษาเขียนโปรแกรมคอมพิวเตอร์วิทัศน์เขียนบัคกันเยอะที่สุดครับ! เวลาเราเรียนคณิตศาสตร์ เราคุ้นชินกับการสะกดพิกัดแบบ (x, y) เสมอ แต่ในทางคอมพิวเตอร์ศาสตร์ เมื่อรูปภาพถูกจัดเก็บอยู่ในรูปแบบแมทริกซ์ของ NumPy ตัวแปรจะถูกจัดเรียงตามระบบ 'แถวและคอลัมน์' (Rows and Columns) แถวของภาพวิ่งในแนวตั้งซึ่งเปรียบเสมือนแกน y ส่วนคอลัมน์วิ่งในแนวนอนเปรียบเสมือนแกน x ดังนั้นจำกฎข้อนี้ให้ขึ้นใจครับ: หากต้องการอ่านหรือดัดแปลงสีพิกเซลตรง ๆ ผ่าน NumPy ต้องเขียนตัว y ขึ้นก่อนแกน x เสมอ ในรูปแบบ `img[y, x]`"*

### สไลด์ที่ 4: ความลึกของบิตภาพ (Bit Depth) และขอบเขตค่าสี 0 - 255
*   **โครงสร้างการจัดวางหน้าสไลด์ (Visual Layout):**
    *   ภาพสเปกตรัมไล่โทนสีจากดำสุด (ระบุเลข 0) ไปจนถึงขาวสุด (ระบุเลข 255)
    *   สูตรคำนวณคณิตศาสตร์: $2^8 = 256$ ระดับสี
*   **เนื้อหาบนสไลด์ (Slide Content):**
    *   **บิตดีพธ์ (Bit Depth):** ตัวระบุว่าใน 1 พิกเซลจะเก็บรายละเอียดความสว่างได้ละเอียดแค่ไหน
    *   ภาพดิจิทัลทั่วไปใช้ระบบภาพ **8 บิต (8-bit Image)**
    *   ค่าความสว่างที่เป็นไปได้คือ $2^8 = 256$ ค่า (ช่วงตัวเลขตั้งแต่ **0 ถึง 255**)
        *   `0` = สีดำสนิท (มืดที่สุด ไม่สะท้อนแสง)
        *   `255` = สีขาวสนิท (สว่างที่สุด แสงเต็มช่องแชนเนล)
    *   ประเภทข้อมูลพิกเซลใน Python: **`np.uint8`** (Unsigned Integer 8-bit หรือจำนวนเต็มไม่มีเครื่องหมายขนาด 8 บิต)
*   **บทบรรยายประกอบการสอนอย่างละเอียด (Teacher's Script & Review Note):**
    *   *"ทำไมค่าสีต้องหยุดอยู่ที่ 255? เพราะในระบบคอมพิวเตอร์ 8 บิต คือหน่วยความจำขนาด 1 ไบต์ (Byte) ตัวเลขฐานสองที่สามารถเก็บได้สูงสุดใน 8 บิตคือ 11111111 ซึ่งแปลงเป็นฐานสิบได้ 255 นั่นเองครับ หากเราคำนวณบวกเลขพิกเซลแล้วได้ค่าเกิน 255 (เช่น 260) ในชนิดข้อมูล uint8 ค่าจะเกิดการล้นลูปพิกเซล (Overflow) วนกลับไปที่เลขน้อย ๆ ทำให้พิกเซลที่ควรจะสว่างกลายเป็นสีมืดทันที ในทำนองเดียวกันถ้าได้ค่าเป็นลบ (เช่น -5) ค่าจะล้นรอบกลับไปที่ 250 ซึ่งเป็นค่าสว่าง นี่คือสาเหตุที่เราต้องระวังในการปรับความสว่างภาพแบบคณิตศาสตร์"*

### สไลด์ที่ 5: การเข้าถึงเฉพาะส่วนด้วย NumPy Slicing (การหั่นตารางพิกเซล)
*   **โครงสร้างการจัดวางหน้าสไลด์ (Visual Layout):**
    *   ภาพถ่ายแมวตัวหนึ่งขนาดใหญ่
    *   มีกรอบสี่เหลี่ยมโปร่งใสสีแดงตีกรอบบริเวณใบหน้าของแมว
    *   ชี้ลูกศรนำภาพหน้าแมวที่ถูกหั่นออกมาวางแยก แสดงสูตรการเขียนโค้ด:
        `roi = img[y_start:y_end, x_start:x_end]`
*   **เนื้อหาบนสไลด์ (Slide Content):**
    *   **ROI (Region of Interest):** พื้นที่เป้าหมายภายในรูปภาพที่เราต้องการสนใจเป็นพิเศษ
    *   เราไม่จำเป็นต้องใช้ฟังก์ชันซับซ้อนในการดึงภาพบางส่วนออกมา เพียงใช้ **NumPy Slicing**
    *   โครงสร้างคำสั่ง:
        ```python
        cropped = img[y_เริ่ม:y_จบ, x_เริ่ม:x_จบ]
        ```
*   **บทบรรยายประกอบการสอนอย่างละเอียด (Teacher's Script & Review Note):**
    *   *"คำว่า Slicing แปลว่า การหั่นเป็นแผ่น ๆ ครับ ลองจินตนาการว่าภาพถ่ายคือขนมปังแถวใหญ่ ถ้าเราต้องการหน้าของน้องแมวที่อยู่กลางรูป เราต้องใช้คำสั่งหั่นพิกัด แถวตั้งเริ่มต้นจนถึงแถวตั้งสิ้นสุด (ช่วงแกน y) และคอลัมน์แนวนอนเริ่มต้นจนถึงคอลัมน์แนวนอนสิ้นสุด (ช่วงแกน x) ตัวอย่างเช่น `img[100:300, 150:450]` จะเป็นการหยิบพิกเซลตั้งแต่แถว 100 ถึง 299 และคอลัมน์ 150 ถึง 449 มาสร้างเป็นรูปภาพใหม่ วิธีการนี้ทำงานเร็วมากและประหยัดแรมเพราะไม่มีการคัดลอกพิกเซลทั้งหมดใหม่"*

### สไลด์ที่ 9: อธิบายพารามิเตอร์ Interpolation แต่ละแบบ (เมื่อไหร่ควรใช้ตัวไหน?)
*   **โครงสร้างการจัดวางหน้าสไลด์ (Visual Layout):**
    *   ตารางเปรียบเทียบผลลัพธ์รูปภาพผลไม้นำมาขยายและย่อด้วย 3 วิธีที่แตกต่างกันเด่นชัด เช่น ภาพที่ใช้วิธี `NEAREST` จะแตกรอยหยัก ในขณะที่วิธี `CUBIC` จะดูสมูทนุ่มนวล
*   **เนื้อหาบนสไลด์ (Slide Content):**
    *   **`cv2.INTER_NEAREST`** (มองเพื่อนบ้านใกล้สุด): เร็วสุด มีการก็อปปี้ค่าสีที่อยู่ใกล้ที่สุดมาใส่ทันที ภาพผลลัพธ์จะหยัก แตกเป็นบล็อกเหลี่ยม
    *   **`cv2.INTER_LINEAR`** (เชิงเส้นคู่): เป็นค่ามาตรฐาน ทำงานค่อนข้างเร็ว ความคมชัดกลาง ๆ เหมาะสำหรับสเกลภาพขยายขึ้นทั่วไป
    *   **`cv2.INTER_AREA`** (คำนวณพื้นที่เฉลี่ย): **จำเป็นต้องใช้ทุกครั้งเมื่อทำการย่อรูปภาพให้เล็กลง** เพื่อป้องกันไม่ให้เกิดความสั่นไหวของเส้นขอบวัตถุ
    *   **`cv2.INTER_CUBIC`** (ลูกบาศก์คู่): ใช้พื้นที่ 4x4 พิกเซลในการวิเคราะห์ความลาดเอียงสี เหมาะกับการขยายภาพให้คมชัดสูง
*   **บทบรรยายประกอบการสอนอย่างละเอียด (Teacher's Script & Review Note):**
    *   *"จำหลักเกณฑ์ง่าย ๆ ไปใช้ตลอดการทำงานสายคอมพิวเตอร์วิทัศน์เลยนะครับเด็ก ๆ:
        1. **ถ้าจะย่อภาพ** ให้เล็กลง (เช่น ข้อมูลชุดเดิมขนาดใหญ่มาก ย่อมาเป็น 224x224 ป้อน AI) -> ใช้ `cv2.INTER_AREA` เสมอ
        2. **ถ้าจะขยายภาพ** ให้ใหญ่ขึ้นเพื่อนำไปใช้แสดงผลสวย ๆ -> ใช้ `cv2.INTER_CUBIC` หรือ `cv2.INTER_LANCZOS4` เพื่อภาพนุ่มนวลที่สุด"*

### สไลด์ที่ 10: การกลับด้านภาพ (Flipping) และรหัส Flip Code
*   **โครงสร้างการจัดวางหน้าสไลด์ (Visual Layout):**
    *   ภาพต้นฉบับรูปหน้าคนมองไปทางซ้าย
    *   ภาพ 3 ผลลัพธ์จากการใช้รหัสต่างกันชี้ออกจากรูปต้นฉบับ:
        *   ชี้ไปที่ `flipCode = 1`: หน้าคนสลับข้างมองไปทางขวา (Horizontal Mirror)
        *   ชี้ไปที่ `flipCode = 0`: คนกลับหัวลงดิน (Vertical Upside Down)
        *   ชี้ไปที่ `flipCode = -1`: คนกลับหัวและสลับซ้ายขวา
*   **เนื้อหาบนสไลด์ (Slide Content):**
    *   การสะท้อนกลับพิกเซลตามแกนตั้งหรือแกนนอนด้วยคำสั่ง:
        ```python
        flipped = cv2.flip(src, flipCode)
        ```
    *   **รหัสการประมวลผล (Flip Code):**
        *   `1` = กลับด้านแนวนอน (Horizontal Flip) - พลิกกระจกตามแกน Y
        *   `0` = กลับด้านแนวตั้ง (Vertical Flip) - พลิกคว่ำบนล่างตามแกน X
        *   `-1` = กลับด้านภาพทั้งแกน X และแกน Y พร้อมกัน
*   **บทบรรยายประกอบการสอนอย่างละเอียด (Teacher's Script & Review Note):**
    *   *"การกลับด้านภาพเป็นเทคนิคยอดนิยมในการฝึกสอนโมเดลปัญญาประดิษฐ์ครับ ลองนึกภาพกล้องหน้ารถยนต์ที่วิ่งบนถนนเมืองไทยพวงมาลัยขวา กับกล้องหน้ารถยนต์ต่างประเทศพวงมาลัยซ้าย หากเราเขียนโปรแกรมดึงรูปมาทำ Flip แนวนอน (flipCode=1) เราจะได้รูปภาพการจราจรแบบเลนกลับกันทันที ช่วยเพิ่มความหลากหลายของรูปแบบข้อมูลโดยไม่ต้องไปถ่ายรูปภาพใหม่เลย"*

### สไลด์ที่ 11: การหมุนภาพมุมตรง (Rotate 90/180) vs การบิดมุมเอียงด้วย Affine Matrix
*   **โครงสร้างการจัดวางหน้าสไลด์ (Visual Layout):**
    *   ฝั่งซ้าย: โชว์การหมุนภาพ 90 องศาตรง ๆ ด้วยฟังก์ชัน `cv2.rotate` (ไม่มีขอบดำแหว่งเกิดขึ้น)
    *   ฝั่งขวา: โชว์การหมุนภาพเอียง 15 องศาด้วย `cv2.warpAffine` แสดงขอบดำของภาพที่แหว่งไปและพิกัดการหาจุดกึ่งกลางภาพ
*   **เนื้อหาบนสไลด์ (Slide Content):**
    *   **หมุนมุมฉากแบบรวดเร็ว (ไม่ต้องคำนวณแมทริกซ์):**
        ```python
        img_90 = cv2.rotate(img, cv2.ROTATE_90_CLOCKWISE)
        ```
    *   **หมุนมุมอิสระ (เช่น เอียง 15 องศา):** ต้องอาศัยการย้ายพิกัดเชิงเส้น (Affine Transform)
        1. หาแมทริกซ์การหมุนรอบจุดกึ่งกลาง:
           ```python
           M = cv2.getRotationMatrix2D(center, angle_deg, scale)
           ```
        2. นำแมทริกซ์การหมุน $M$ ไปบิดพิกัดภาพ:
           ```python
           rotated = cv2.warpAffine(img, M, (width, height))
           ```
*   **บทบรรยายประกอบการสอนอย่างละเอียด (Teacher's Script & Review Note):**
    *   *"การหมุนทั่วไปเป็นแบบทำมุมฉาก ซึ่งคอมพิวเตอร์แค่จับพิกเซลสลับแกนคอลัมน์กันตรง ๆ จึงเร็วมากครับ แต่หากเราอยากหมุนเอียง 15 องศา เพื่อเลียนแบบการจับภาพวัตถุในกล้องบนสายพานที่วางไม่ตรงแนว เราต้องใช้คณิตศาสตร์แคลคูลัสเวกเตอร์แอฟฟีนแมทริกซ์เข้ามาช่วยแปลงพิกัดของแต่ละจุดในแมทริกซ์ภาพครับ โค้ดจะเริ่มมีสองจังหวะคือ 1. สร้างสูตรแมทริกซ์การย้ายค่า ($M$) และ 2. นำรูปภาพมาผ่านสูตรการย้ายพิกัดนั้นด้วยฟังก์ชัน `warpAffine` โดยช่องว่างมุมที่หมุนแล้วไม่มีข้อมูลจะถูกระบายสีดำ (`borderValue=(0, 0, 0)`) แทน"*

### สไลด์ที่ 13: ภาพรวมระบบสคริปต์ประมวลผลเป็นกลุ่ม (Batch Processing Pipeline)
*   **โครงสร้างการจัดวางหน้าสไลด์ (Visual Layout):**
    *   Flowchart ลำดับการทำงานตั้งแต่โฟลเดอร์เริ่มต้น `dataset_raw` วนลูปอ่านรูปภาพ -> ตกแต่งแปลงขนาด/กลับด้าน/หมุนภาพ -> บันทึกอัตโนมัติลงในโฟลเดอร์ปลายทาง `dataset_augmented`
*   **เนื้อหาบนสไลด์ (Slide Content):**
    *   การทำงานกับข้อมูลรูปภาพหลักพันรูปเป็นไปไม่ได้ที่จะทำมือทีละภาพ
    *   **การเขียนสคริปต์ประมวลผลกลุ่ม (Batch Script):**
        *   ตรวจสอบความมีอยู่ของโฟลเดอร์นำเข้า/ส่งออกด้วยไลบรารี `os`
        *   สแกนอ่านไฟล์รูปภาพนามสกุลยอดนิยมในโฟลเดอร์ผ่าน `glob`
        *   รวบรวมฟังก์ชันเรขาคณิต (Resize, Flip, Rotate, Crop) มารันวนลูปและส่งออกไฟล์ภาพอัตโนมัติ
*   **บทบรรยายประกอบการสอนอย่างละเอียด (Teacher's Script & Review Note):**
    *   *"ในส่วนปฏิบัติการแล็บสัปดาห์นี้ นักศึกษาจะเรียนรู้วิธีการเขียนสคริปต์อัตโนมัติ 1 สคริปต์ ซึ่งเมื่อรันแล้วระบบจะทำการกวาดรูปภาพทั้งหมดที่เราโยนไว้ในโฟลเดอร์แรก แล้วประมวลผลตามแผนภาพนี้ทันทีเพื่อออกลูกออกมาเป็นรูปภาพเวอร์ชันปรับแต่งที่หลากหลาย นี่คือทักษะการเป็น Software Engineer ด้าน Computer Vision ตัวจริงที่สามารถจัดการกับปริมาณข้อมูลขนาดใหญ่ได้โดยอัตโนมัติครับ"*
