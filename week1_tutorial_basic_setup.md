### ขั้นตอนที่ 1: ติดตั้งโปรแกรมหลัก (VS Code & Miniconda)
1. **ติดตั้ง Visual Studio Code (VS Code)**
   * ดาวน์โหลดและติดตั้งตามระบบปฏิบัติการของนักศึกษาจาก [code.visualstudio.com](https://code.visualstudio.com/)
2. **ติดตั้ง Miniconda (ตัวจัดการ Environment และแพ็คเกจ)**
   * หลีกเลี่ยงการดาวน์โหลด Python ตัวเต็มธรรมดาเพื่อป้องกันปัญหาความเสถียร
   * เข้าไปที่ [docs.conda.io/en/latest/miniconda.html](https://docs.conda.io/en/latest/miniconda.html)
   * ดาวน์โหลดเวอร์ชันล่าสุดสำหรับ Windows, macOS หรือ Linux
   * **สำคัญสำหรับผู้ใช้ Windows**: ในหน้าต่างเลือกตั้งค่าการติดตั้ง ให้ทำเครื่องหมายถูกที่ช่อง **"Add Miniconda3 to my PATH environment variable"** (หรือทำเครื่องหมายถูกในช่องที่แนะนำทั้งหมด) เพื่อให้สามารถรันคำสั่ง conda ผ่าน Terminal ได้สะดวก

### ขั้นตอนที่ 3: สร้างโฟลเดอร์โครงงานและสร้างไฟล์คอนฟิกสภาพแวดล้อม
1. สร้างโฟลเดอร์สำหรับวิชานี้บนเครื่องคอมพิวเตอร์ของคุณ เช่น `D:\Image_Processing_Class`
2. เปิดโปรแกรม VS Code ขึ้นมาแล้วไปที่เมนู **File -> Open Folder...** จากนั้นคลิกเลือกเปิดโฟลเดอร์ที่เพิ่งสร้างขึ้นมา
3. คลิกขวาที่พื้นที่ว่างในแถบ Explorer ด้านซ้าย เลือก **New File** แล้วตั้งชื่อไฟล์ว่า `environment.yml`
4. คัดลอกข้อความด้านล่างนี้ไปวางลงในไฟล์ `environment.yml` แล้วกดบันทึก (`Ctrl + S`):

```yaml
name: dip_env
channels:
  - pytorch
  - conda-forge
  - defaults
dependencies:
  - python=3.10.12
  - numpy=1.24.3
  - matplotlib=3.7.1
  - opencv=4.6.0
  - pytorch=2.0.1
  - torchvision=0.15.2
  - cpuonly
  - pip
  - pip:
    - ultralytics==8.0.196
    - mediapipe==0.10.7
```

### ขั้นตอนที่ 4: สร้าง Environment และเปิดใช้งานผ่าน VS Code Terminal
1. เปิด Terminal ใน VS Code โดยไปที่เมนู **Terminal -> New Terminal** (หรือกดปุ่ม `Ctrl + Shift + \``)
2. ในช่อง Terminal (หน้าต่างสีดำด้านล่าง) พิมพ์คำสั่งนี้เพื่อสร้างสภาพแวดล้อมจำลองตามที่กำหนดในไฟล์ `environment.yml`:
   ```bash
   conda env create -f environment.yml
   ```
   *(ขั้นตอนนี้จะใช้เวลาประมาณ 3-5 นาที ระบบจะดาวน์โหลด Python เวอร์ชัน 3.10.12 และไลบรารีทั้งหมดที่จำเป็นตามความลึกที่เหมาะสม)*
3. เมื่อระบบดาวน์โหลดเสร็จสิ้น ให้เปิดใช้งานสภาพแวดล้อมจำลองด้วยคำสั่ง:
   ```bash
   conda activate dip_env
   ```
4. **การตั้งค่า Interpreter ใน VS Code**:
   * กดปุ่มทางลัด `Ctrl + Shift + P` เพื่อเปิดหน้าต่างค้นหาคำสั่ง (Command Palette)
   * พิมพ์ค้นหาคำว่า `Python: Select Interpreter` แล้วกด Enter
   * ระบบจะแสดงรายชื่อ Python ในเครื่องขึ้นมา ให้คลิกเลือกตัวที่มีชื่อว่า **`Python 3.10.12 ('dip_env': conda)`**
   *(หากไม่พบรายชื่อ ให้กดปุ่มรีเฟรช หรือปิด VS Code แล้วเปิดโฟลเดอร์ขึ้นมาใหม่อีกครั้ง)*

## ส่วนที่ 2: ทฤษฎีภาพดิจิทัลเบื้องต้น (0.5 ชั่วโมง)

ก่อนจะเริ่มลงมือโค้ดจริง มาทำความเข้าใจทฤษฎีภาพพื้นฐาน 4 ข้อนี้:
1. **ภาพคือแมทริกซ์ 2 หรือ 3 มิติ**:
   * ภาพขาวดำ/เกรย์สเกล (Grayscale) คือตารางตัวเลข 2 มิติ (กว้าง x สูง) แต่ละพิกเซลเก็บค่าความสว่างตั้งแต่ `0` (สีดำสนิท) ไปจนถึง `255` (สีขาวสนิท)
   * ภาพสี (BGR) คือแมทริกซ์ 3 มิติ (กว้าง x สูง x 3 แชนเนลสี) โดยใน OpenCV จะเรียงลำดับแชนเนลเป็น **Blue, Green และ Red** แตกต่างจากโปรแกรมอื่นที่เรียงแบบ RGB
2. **ขอบเขตพิกัดภาพ (Coordinate System)**:
   * พิกัดจุดเริ่มตนน $(x=0, y=0)$ จะอยู่ที่ **"มุมซ้ายบนสุดของภาพ"**
   * ค่า $x$ วิ่งไปทางขวา (ความกว้าง) และค่า $y$ วิ่งลงข้างล่าง (ความสูง)
3. **ความลึกบิตภาพ (Bit Depth)**:
   * ภาพที่เราใช้งานทั่วไปเป็นภาพ 8-บิต (8-bit) ซึ่งหมายความว่าในหนึ่งจุดพิกเซลจะมีค่าความสว่างที่เป็นไปได้ทั้งหมด $2^8 = 256$ ระดับ (ค่าตั้งแต่ 0 ถึง 255)

## ส่วนที่ 3: โค้ดแล็บปฏิบัติการแรก - Image Input/Output (1 ชั่วโมง)

### ขั้นตอนที่ 5: การสร้างสคริปต์เปิดและแปลงรูปภาพ
1. เตรียมรูปภาพตัวอย่าง 1 ภาพในโฟลเดอร์งาน ตั้งชื่อว่า `test_image.jpg`
2. สร้างไฟล์ใหม่ใน VS Code ชื่อว่า `main.py`
3. เขียนสคริปต์ Python ลงในไฟล์ `main.py` ตามตัวอย่างด้านล่างนี้:

```python
# main.py
import cv2
import numpy as np

def main():
    print("--- เริ่มต้นการโหลดรูปภาพ ---")
    
    # 1. โหลดรูปภาพต้นแบบ (OpenCV จะเก็บในฟอร์แมตสี BGR)
    # ฟังก์ชัน cv2.imread(เส้นทางไฟล์, พารามิเตอร์)
    img_color = cv2.imread("test_image.jpg")
    
    # ตรวจสอบว่าเปิดไฟล์ภาพได้หรือไม่
    if img_color is None:
        print("ข้อผิดพลาด: ไม่พบไฟล์รูปภาพ 'test_image.jpg' กรุณาตรวจสอบตำแหน่งไฟล์!")
        return

    # 2. ตรวจสอบข้อมูลพิกเซลด้วย NumPy
    # .shape จะบอกขนาดภาพในรูปแบบ (ความสูง, ความกว้าง, แชนเนลสี)
    height, width, channels = img_color.shape
    print(f"มิติของรูปภาพสี: {width} x {height} พิกเซล")
    print(f"จำนวนแชนเนลสี: {channels} แชนเนล (BGR)")
    print(f"ชนิดข้อมูลพิกเซล: {img_color.dtype}") # มักเป็น uint8 (ตัวเลข 0-255)

    # 3. แปลงภาพสี (BGR) เป็นภาพขาวดำเกรย์สเกล (Grayscale)
    # ใช้ฟังก์ชัน cv2.cvtColor()
    img_gray = cv2.cvtColor(img_color, cv2.COLOR_BGR2GRAY)
    print(f"มิติของรูปภาพขาวดำเกรย์สเกล: {img_gray.shape} (ไม่มีแชนเนลสีที่ 3)")

    # 4. แสดงผลลัพธ์ภาพบนหน้าต่างโปรแกรม
    # cv2.imshow('ชื่อหน้าต่าง', ตัวแปรภาพ)
    cv2.imshow("Original Color Image", img_color)
    cv2.imshow("Grayscale Imageed", img_gray)

    # 5. สั่งบันทึกภาพเกรย์สเกลลงเครื่องคอมพิวเตอร์เป็นไฟล์รูปภาพใหม่
    # cv2.imwrite('เส้นทางบันทึก', ตัวแปรภาพ)
    cv2.imwrite("output_grayscale.png", img_gray)
    print("ระบบบันทึกรูปภาพสีขาวดำเสร็จสิ้นในชื่อ 'output_grayscale.png'")

    # 6. คำสั่งค้างหน้าต่างแสดงผลไว้จนกว่าจะกดปุ่มใด ๆ บนแป้นพิมพ์
    print("กรุณากดปุ่มใด ๆ บนแป้นพิมพ์เพื่อปิดหน้าต่างภาพ...")
    cv2.waitKey(0) 
    
    # สั่งล้างหน่วยความจำและปิดหน้าต่างทั้งหมดของ OpenCV
    cv2.destroyAllWindows()
    print("ปิดหน้าต่างเรียบร้อยแล้ว!")

if __name__ == "__main__":
    main()
```

### ขั้นตอนที่ 6: วิธีการทดสอบรันโปรแกรมใน VS Code
มี 2 วิธีหลักในการสั่งรันสคริปต์ใน VS Code:
* **วิธีที่ 1 (ผ่านปุ่มลัด)**: คลิกที่แท็บไฟล์ `main.py` จากนั้นกดปุ่มสามเหลี่ยมสีดำ (Run Python File) ที่มุมขวาบนของหน้าต่าง VS Code
* **วิธีที่ 2 (ผ่าน Terminal)**: ตรวจสอบว่า Terminal ขึ้นคำว่า `(dip_env)` นำหน้าบรรทัดคำสั่งหรือไม่ ถ้าขึ้นเรียบร้อยแล้ว ให้พิมพ์คำสั่งเพื่อสั่งรันโปรแกรม:
  ```bash
  python main.py
  ```

> [!WARNING]
> เมื่อรันโปรแกรมสำเร็จ หน้าต่างแสดงภาพจะเด้งขึ้นมา **ห้ามใช้นิ้วไปคลิกปุ่มปิด [X] ที่ตัวหน้าต่างภาพโดยตรง** เพราะจะทำให้โปรแกรมค้าง ให้กดปุ่มใด ๆ บนคีย์บอร์ด (เช่น Spacebar หรือ Enter) เพื่อเปิดโอกาสให้ฟังก์ชัน `cv2.waitKey(0)` และ `cv2.destroyAllWindows()` ทำความสะอาดหน่วยความจำของโปรแกรมให้เรียบร้อย
